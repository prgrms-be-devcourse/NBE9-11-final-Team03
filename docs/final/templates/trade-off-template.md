## 작성 예시

### 제목

`[S3] 서버 로컬 저장 vs S3 객체 저장`

### 1. 결정 배경

| 항목 | 내용 |
| --- | --- |
| 관련 기능 | S3 / 결과물 제출 / 재능 첨부 파일 |
| 결정 시점 | 2026-06-21 |
| 우선순위 | P1 |
| 결정자 |  |
| 목적 | 결과물 제출과 파일 첨부 기능 확장을 위해 파일 저장 방식을 결정 |

### 2. 선택지 비교

| 선택지 | 장점 | 단점 | 리스크 |
| --- | --- | --- | --- |
| 서버 로컬 저장 | 구현이 단순하고 빠름 | 배포 환경 변경 시 파일 유지가 어려움 | 컨테이너 재시작/교체 시 파일 유실 가능 |
| DB BLOB 저장 | 트랜잭션 관리가 쉬움 | DB 용량 증가, 응답 성능 저하 가능 | 파일이 커질수록 DB 운영 부담 증가 |
| S3 객체 저장 | 서버와 파일 저장소 분리, 확장성 높음 | AWS 설정과 권한 관리 필요 | presigned URL 만료, 권한 설정 실수 가능 |

### 3. 최종 선택

| 항목 | 내용 |
| --- | --- |
| 선택한 방식 | S3 객체 저장 |
| 선택 이유 | 결과물 제출, 재능 첨부 파일, 프로필 이미지 등 파일 기능 확장에 유리 |
| 포기한 것 | 로컬 저장 방식의 단순함 |
| 감수한 리스크 | AWS 설정, 비용, 권한 관리, 테스트 환경 분리 필요 |
