# 2026-06-21 Baton 회의 안건

오늘 회의의 목표는 진행 상황 공유가 아니라, **PURCHASE MVP 완성을 위한 남은 작업과 담당자를 확정하는 것**입니다.

회의가 끝났을 때 아래 3가지는 반드시 정해져 있어야 합니다.

- MVP P0 작업 담당자
- 오늘/내일 완료 기준
- MVP 이후 고도화 우선순위

- S3 공통 코드: 머지 완료

- 매칭 제안 생성/수락/거절: 부분 구현

현재 가장 큰 문제는 **기능 조각은 있는데 PURCHASE 흐름이 끝까지 이어지지 않는 것**입니다.

## 2. MVP P0 안건

### 안건 1. 회원가입 시 초기 크레딧 지급

- 관련 Jira: `BATON-48`
- 현재 상태: `CreditService.initializeAccount()`는 있으나 회원가입 흐름에 연결 필요
- 결정할 것:
  - 누가 맡을지
  - 오늘 어디까지 끝낼지
  - 회원가입 후 잔액 조회까지 확인할지

추천 담당:

- 이유진
- 필요 시 최윤서 지원

완료 기준:

- 회원가입 성공
- `CreditAccount` 생성
- 초기 크레딧 지급
- WELCOME 크레딧 내역 기록

## 3. MVP P0 안건

### 안건 2. 매칭 수락 후 PURCHASE 거래 연결

- 관련 Jira: `BATON-79`
- 현재 상태: 매칭 수락 시 `ACCEPTED` 상태 변경은 가능하지만, Trade/Escrow/Credit 연결 필요
- 결정할 것:
  - 이인희/최윤서가 어디까지 같이 볼지
  - Trade 생성과 Credit hold를 어느 서비스에서 묶을지
  - 실패 시 ACCEPTED로 바뀌지 않게 처리할지

추천 담당:

- 이인희
- 최윤서

완료 기준:

- 매칭 제안 수락
- PURCHASE `Trade` 생성
- `Escrow` 생성
- 구매자 크레딧 에스크로 보류
- ESCROW_HOLD 내역 기록

## 4. MVP P0 안건

### 안건 3. 거래 완료/구매 확정/제공자 정산

- 관련 Jira: `BATON-72`
- 현재 상태: 거래 조회/취소는 있으나 구매 확정과 정산 API 필요
- 결정할 것:
  - MVP에서 수동 구매 확정만 할지
  - Auto-Release/Dispute는 후속으로 뺄지
  - 정산 내역 기록까지 포함할지

추천 담당:

- 최윤서

완료 기준:

- 구매 확정 API 호출
- Trade 상태 `COMPLETED`
- Escrow 상태 `RELEASED`
- 구매자 escrowBalance 감소
- 제공자 balance 증가
- ESCROW_RELEASE 내역 기록

## 6. 고도화 안건

### 안건 5. MVP 이후 추가 기능 우선순위

고도화는 반드시 진행하되, MVP 흐름과 분리해서 진행합니다.

추천 우선순위:

1. 재능 첨부파일
   - 현재 열린 PR `#53`
   - S3 기반이 이미 머지되어 자연스럽게 이어갈 수 있음

2. 리뷰/신뢰 점수
   - 거래 완료 후 연결하기 좋음
   - Baton 서비스 차별화에 도움

3. 관리자 기능
   - 사용자/재능/거래 관리
   - 정책 문서와 연결하기 좋음

4. 채팅
   - 실시간보다 메시지 저장/조회부터 시작

5. SWAP
   - 서비스 비전에는 중요하지만 PURCHASE 안정화 후 진행
